ドナルド・クヌースの『The Art of Computer Programming』
（TAOCP、1968年〜）は、計算機科学における最も野心的な
著作である。全7巻（うち3巻と数個の分冊が刊行済み）で
アルゴリズムの百科全書を目指すこの仕事は、タイトルが
示すとおり、プログラミングを「技術（technology）」ではなく
「技芸（art）」として位置づける。

クヌースはこう述べている：

  "We have seen that computer programming is an art,
   because it applies accumulated knowledge to the world,
   because it requires skill and ingenuity, and especially
   because it produces objects of beauty."

  （プログラミングが芸術であるのは、蓄積された知識を
   世界に適用するからであり、技能と創意を要するから
   であり、とりわけ美しいオブジェクトを生み出すからである。）

「美しい」アルゴリズムとは何か？ クヌースにとって、
アルゴリズムの美しさは以下の要素からなる：
  ・簡潔性（不要な複雑さがない）
  ・意外性（予想外のアイデアが使われている）
  ・必然性（その構造であるべき必然性がある）
  ・一般性（特殊な場合だけでなく広く適用できる）

これは、詩の美しさの基準と驚くほど重なる：
  ・簡潔性（一語も余分でない）→ ポーの「短詩論」
  ・意外性（予想を裏切る比喩、展開）→ 異化（→§16）
  ・必然性（他の表現ではありえない）→ フローベールの「ぴったりの語」
  ・一般性（個別を超えて普遍に至る）→ アリストテレスの詩学（→§15）

クヌースはまた【文芸的プログラミング（Literate Programming）】
（1984年）の提唱者でもある。文芸的プログラミングとは、
プログラムを「まず人間のために書き、次に機械のために書く」
手法である。プログラムの論理を自然言語で説明する文章の
中にコードが埋め込まれ、文書全体が読み物として成立する。

これは、Code Poetry（→§31）の精神と深く共鳴する。
Code Poetryが「プログラムを詩として読む」ことを目指す
のに対し、文芸的プログラミングは「プログラムをエッセイ
として読む」ことを目指す。両者に共通するのは、「コードの
可読性」を「人間にとっての意味」の観点から追求する姿勢である。

ポール・グレアムは「ハッカーと画家」（2003年）で、
プログラミングを絵画制作と比較している。両者とも
「最初は曖昧な全体像から始め、徐々に詳細を
書き込んでいく」プロセスを共有する。プログラマーも
画家も職人（craftsperson）であり、素材を手で操作する
感覚を持つ。Knuthの「art」、グレアムの「painting」——
計算はつねに、自らを芸術の言葉で語ろうとする衝動を
持っている。

情報学的にKnuthの仕事が示すのは、アルゴリズム設計が
単なる工学的問題解決ではなく、【審美的判断を伴う創造行為】
であるということだ。ソートアルゴリズムは数十種類あり、
すべて正しく動作するが、「美しい」ものはごくわずかである。
クイックソートの「美」は、単純な再帰的分割が平均的に
最速の整列を実現するという意外性にある。
