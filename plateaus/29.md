1990年代半ば、インターネットの黎明期に
JODI（Joan Heemskerk と Dirk Paesmans のデュオ）は、
ウェブサイト自体を芸術作品とする「ネットアート」の
先駆けとなった。彼らの作品 wwwwwwwww.jodi.org は、
HTMLソースコードに隠された図像と、意図的に「壊れた」
インターフェースで構成される。

JODIの作品において重要なのは、ソースコードの
「読解」が作品体験の本質的な部分であることだ。
表示されるページは混沌としたテキストの洪水だが、
「ソースを見る（View Source）」で初めて
ASCII アートが姿を現す。これは「テクストの背後の
テクスト」——人間が読むテクストの背後に機械が読む
テクストがある——という二重構造の芸術化であり、
Code Poetry（→§31）の先駆的実践と見ることができる。

ソフトウェアアートの概念は2001年の Transmediale 
フェスティバルで明確に打ち出された。ソフトウェアアートとは、
ソフトウェア——コードとその実行——そのものを
芸術の素材とする実践である。アレクサンダー・ギャロウェイは
『Protocol: How Control Exists after Decentralization』
（2004年）で、コードが文化的テクストとして読解可能な
対象であることを論じた。

Casey Reas と Ben Fry が開発した【Processing】（2001年）は、
ソフトウェアアートの民主化において決定的な役割を果たした。
Processing は「プログラミングを知らない芸術家」が
コードで作品を作るための言語・環境であり、
setup() と draw() という二つの関数が基本構造を成す。
draw()関数はフレームごとに繰り返し実行される
——これはチューリングマシンの「ステップごとの実行」の
視覚化された形態と言える。

Processing から派生した p5.js（JavaScript版）、
openFrameworks（C++のクリエイティブコーディング
ライブラリ）、TouchDesigner（ノードベースの
ビジュアルプログラミング環境）は、いずれも
「コードで作る芸術」のエコシステムを拡張している。

Lev Manovich の『Software Takes Command』（2013年）は、
ソフトウェアが「メタ・メディウム」——すべてのメディアを
シミュレートし、さらに新しいメディアを発明する
メディウム——であると論じた。これは万能チューリングマシン
（→§8 フォン・ノイマン）の文化理論的な読み替えである：
あらゆるメディアをシミュレートできる「万能メディウム」
としてのコンピュータ。
