Code Poetry（コード詩）とは、プログラミング言語で書かれた
テクストが「コード」と「詩」の両方として読めるものである。
この【二重可読性（dual readability）】こそがCode Poetryの
本質的な美的原理である。

Code Poetryの系譜：

  【Perl Poetry】（1990年代）：Perlは自然言語に近い
  構文を許容するプログラミング言語であり、Larry Wall
  （Perlの創始者・言語学者でもある）とSharon Hopkinsらが
  Perl で「読める詩」を書く実践を始めた。

    listen (to, me);
    open yourself, wide;
    join (me, &you);
    die @last;

  この4行はPerlの合法的なコードであると同時に、
  英語の詩としても読める。listen, open, join, die は
  Perl の組み込み関数でありながら、同時に英語の日常語
  である。この二重性は、チューリングマシンのテープ上の
  記号が「意味を持たない形式的対象」であると同時に
  「意味を読み込まれうる記号」でもあるという
  本講義録の核心的問題を具現化している。

  【Mezangelle】：Mez Breeze（オーストラリア）が
  1990年代末から展開するネット上の言語芸術。
  プログラミング言語の構文（括弧、角括弧、スラッシュ）を
  自然言語に挿入し、【コードと肉体の混淆】を表現する。

    wur[d]ship.x" => pray[er.ware]

  これはコンパイル可能なコードではないが、コードの
  視覚的・構造的特徴を詩的素材として使用している。
  Mez の仕事は「コード」と「肉体（flesh）」の境界を
  溶解させる試みであり、計算の脱身体化（→§7）への
  批判的応答と読める。

  【code {poems} アンソロジー】（Ishac Bertran 編、2012年）は、
  C、Ruby、Python、JavaScript、Brainfuckなど多様な言語で
  書かれたCode Poetryを収集した最初のアンソロジーである。
  ここに収録された作品は、コンパイル/インタープリット
  可能なコードであると同時に、人間の読者にとって
  詩的な意味を持つテクストである。

  【Geoff Cox & Alex McLean『Speaking Code』（2013年）】は、
  コードを「パフォーマティブな発話」として理論化した。
  プログラムは「記述」であると同時に「実行」（遂行）
  でもある——これはオースティンの言語行為論
  （事実確認的発言 vs 遂行的発言）の計算機科学版であり、
  コードの「二重性」を哲学的に根拠づける試みである。
