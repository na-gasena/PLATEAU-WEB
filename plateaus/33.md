Esolang（esoteric programming language＝難解プログラミング
言語）は、実用ではなく「計算とは何か？」を問うために
設計されたプログラミング言語である。

【Brainfuck】（Urban Müller, 1993年）は
8つの命令（> < + - . , [ ]）だけで構成される
チューリング完全な言語である。そのソースコードは：

  ++++++++++[>+++++++>++++++++++>+++>+<<<<-]
  >++.>+.+++++++..+++.>++.<<+++++++++++++++.
  >.+++.------.--------.>+.>.

これは "Hello World!" を出力するプログラムである。
Brainfuckの命令セットはチューリングマシンの操作
（テープ上の移動、値の増減、ループ）の最小限の実装であり、
「チューリング完全性に必要な最小限の要素は何か」を
問う思考実験として機能する。

【Piet】（David Morgan-Mar, 2001年）は
ソースコードが抽象画（ピエト・モンドリアンの
絵画へのオマージュ）であるプログラミング言語である。
色のついたブロックの面積と、隣接するブロック間の
色相・明度の差異が命令を決定する。Piet のプログラムは
「見る」ことができるコードであり、「読む」ことの
意味を根本から問い直す。

【Shakespeare Programming Language】（Karl Hasselström
& Jon Åslund, 2001年）は、ソースコードが
シェイクスピア劇のスクリプトの形式をとる言語である。
登場人物が変数に対応し、台詞が演算命令に対応する。
ここでは「プログラムを書くこと」と「戯曲を書くこと」が
文字通り同一の行為になる。

Daniel Temkin は「Esolangs as Art」（2014年頃〜）で、
esolangを芸術実践として理論化した。Temkinによれば、
esolangは「計算の概念的彫刻」であり、計算の特定の
側面を極端にすることで、通常は不可視の前提を
可視化する。これは異化（→§16）の計算機科学版である
——プログラミング言語の「見慣れた」構文を
破壊することで、「プログラミング言語とは何か」を
知覚させる。

esolangは「チューリング完全性の境界」を探る実験でもある。
どこまで言語を単純化しても、あるいは奇妙にしても、
チューリング完全性は保たれるのか？ Brainfuckの8命令、
さらにはBool（2命令でチューリング完全）、
Iota（1つの組み合わせ子だけでチューリング完全）——
これらは計算の【最小表現】を探求する数学的実験であり、
同時に、俳句（→§21）のような極限的圧縮の
計算機科学版でもある。

難読化コード（obfuscated code）の美学も関連する。
IOCCC（International Obfuscated C Code Contest）は、
もっとも読みにくいCプログラムを競うコンテストであり、
受賞作品はしばしば人間には「読めない」が
コンパイラには「読める」テクストである。
この「人間可読性の限界」の探究は、カミングスの
構文破壊（→§20）の計算機科学版と言えるだろう。
