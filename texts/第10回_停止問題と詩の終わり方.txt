================================================================
第10回  停止問題と詩の終わり方
================================================================

■ テーマ

  チューリングの停止問題——あるプログラムが停止するかどうかを
  事前に判定することは不可能である——という定理を丁寧に理解し、
  その構造を「詩がいつ・どのように終わるべきか」という問いと
  重ね合わせる。

■ 到達目標

  ・停止問題の対角線論法による証明を直観的に理解する
  ・「決定不可能性」の意味を把握する
  ・詩の「終わり」の問題を計算理論的な視座から考察する
  ・「終わらないもの」の美学を認識する

================================================================
講義内容
================================================================

1. 停止問題の復習と深化

   第4回で概説した停止問題を、ここで丁寧に証明する。

   問い：
   「プログラムPと入力xが与えられたとき、
    Pがx上で終了（停止）するかどうかを
    判定するプログラムH(P, x)は存在するか？」

   答え：存在しない。

2. 対角線論法による証明

   背理法で証明する。

   仮定：停止判定プログラムHが存在するとする。
         H(P, x) = "停止する"  もし P(x) が停止するなら
         H(P, x) = "停止しない" もし P(x) が停止しないなら

   新しいプログラムDを構成する：
         D(P) =
           if H(P, P) = "停止する" then 無限ループ
           if H(P, P) = "停止しない" then 停止

   Dは、Hの判定結果を「反転」させるプログラムである。

   ここでD(D)を考える——DにD自身を入力する：
   ・もし H(D, D) = "停止する" なら → D(D) は無限ループ → 矛盾
   ・もし H(D, D) = "停止しない" なら → D(D) は停止 → 矛盾

   どちらの場合も矛盾が生じる。
   したがって、仮定が誤りであり、Hは存在しない。  ■

   この証明の構造：
   ・カントールの対角線論法（実数の非可算性の証明）と同構造
   ・ゲーデルの不完全性定理（第7回）と同構造
   ・「自分自身に適用して反転させる」のがすべてに共通する核

3. 決定不可能性の意味するもの

   停止問題が決定不可能であることは、
   計算の「万能性」に根本的な限界があることを意味する。

   チューリング完全なシステムは「すべてを計算できる」が、
   「すべてについて判断できる」わけではない。

   他の決定不可能な問題の例：
   ・プログラムの等価性判定
   ・任意のプログラムが特定の出力を生成するかの判定
   ・ポストの対応問題
   ・ヒルベルトの第10問題（ディオファントス方程式の決定問題）

   メタ理論的な示唆：
   「完全な知識を持つ全能の機械」は原理的に不可能である。
   これは計算理論が示す最も深遠な帰結の一つである。

4. 詩の「終わり」の問題

   詩はいつ、どのように終わるべきか？
   この問題が「決定不可能」であるかもしれないという
   刺激的な可能性を考えてみる。

   詩の終わり方のパターン：
   ・定型による終止：ソネットの14行目、俳句の第3句
     → 「停止が保証されている」プログラム
   ・意味的完結：物語やメッセージの完了
     → 出力が完了したら停止するプログラム
   ・突然の断絶：中途で止まる、未完のまま残される
     → 異常終了、強制終了
   ・フェイドアウト：徐々に消えていく
     → 漸近的に停止に近づくが到達しない
   ・循環・反復：元に戻って繰り返す
     → 無限ループ

   詩の「停止条件」は詩の外部から与えられるのか、
   それとも詩の内部に含まれるのか？

5. 無限に続く詩、終わらないプログラム

   「終わらない」作品の系譜：
   ・ジェイムズ・ジョイス『フィネガンズ・ウェイク』
     最後の文が最初の文に接続する循環構造
   ・ベケット『ゴドーを待ちながら』
     永遠に到来しない終わりを待つ
   ・Nick Montfort「Taroko Gorge」
     無限に生成され続けるプログラム詩

   チューリングマシンの無限ループ：
   void loop () { loop(); }
   停止しないプログラムは「バグ」か、
   それとも「意図」か、あるいは「詩」か？

6. 沈黙による終止 — ジョン・ケージ「4分33秒」

   ケージの「4分33秒」（1952）：
   演奏者は4分33秒のあいだ、一つの音も演奏しない。
   「音楽の終わり」と「沈黙の始まり」の境界の消滅。

   プログラムとしての「4分33秒」：
     wait(273000);  // 4'33" = 273秒
     exit(0);
   
   このプログラムは「何もしないで停止する」。
   しかし、ケージの作品が示すのは、
   「何も演奏しないこと」は「沈黙を演奏すること」であり、
   「何もしないプログラム」は「何もしないことを実行する
    プログラム」であるという逆説である。

   空白テープの上のチューリングマシン——何も読まず、
   何も書かず、ただ停止する。しかしその「停止」が作品である。

    「フェイドアウト」か、「ループ」か？

     今聴いてみよ）。それは「音楽」か？同じことを
