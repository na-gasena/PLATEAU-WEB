================================================================
第12回  身体・声・パフォーマンス — 実行としての詩
================================================================

■ テーマ

  プログラムの「実行（execution）」と詩の「朗読・パフォーマンス」
  の構造的な類比を探る。身体と声が介在するとき、
  記号はどのように「実行」されるのか。

■ 到達目標

  ・「実行」（execution）の概念を計算と詩の両面から理解する
  ・サウンドポエトリーの歴史と実践を知る
  ・フルクサスのイベントスコアとプログラムの類似を認識する
  ・ライブコーディングと即興詩のパラレルを理解する
  ・「インタプリタとしての身体」の概念を考察する

================================================================
講義内容
================================================================

1. 「実行」という概念の二重性

   execution の二つの意味：
   (a) コンピュータ科学：プログラムの実行
       ・ソースコード → コンパイラ/インタプリタ → 動作
   (b) パフォーマンス芸術：楽曲の演奏、詩の朗読、ダンスの遂行
       ・楽譜/台本/振付 → 演奏者/俳優/ダンサー → パフォーマンス

   両者に共通する構造：
   ・テクスト（記述されたもの） → 解釈者 → 実現
   ・テクストは潜在的であり、実行によって現実化する
   ・同じテクストの異なる「実行」が可能

   チューリングマシンの文脈：
   ・遷移表（プログラム）は「楽譜」
   ・チューリングマシンの動作は「演奏」
   ・テープの状態変化は「パフォーマンス」

2. サウンドポエトリー — 声としての記号

   サウンドポエトリーは、言語の「音」を前景化する詩の伝統。
   言葉の意味を排除あるいは周縁化し、
   音声そのものの物質性に焦点を当てる。

   歴史的展開：
   ・Hugo Ball「Karawane」（1916, ダダ）：
     "gadji beri bimba glandridi laula lonni cadori..."
     意味を持たない音節の羅列。ダダイズムの根源的反抗。

   ・クルト・シュヴィッタース「ウルソナーテ」（1922-1932）：
     約35分のソナタ形式による音声詩。
     音楽のソナタ構造を「言葉の代わりに音素」で実行。
     楽譜（テクスト）が存在し、演奏者はそれを「実行」する。

   ・Henri Chopin — テープレコーダーによる音声詩
     身体の音（呼吸、嚥下）を録音・加工する
     テープ＝チューリングマシンのテープ！
     録音テープの操作＝テープ上の記号操作

   ・Bernard Heidsieck — アクション詩

   サウンドポエトリーとチューリングマシン：
   ・音声記号は意味を持たない → 形式的記号体系
   ・音素の操作（反復・変形・逆行）→ テープ操作
   ・演奏というexecution → プログラムのexecution

3. フルクサスのイベントスコア

   フルクサス（Fluxus, 1960年代〜）は、
   芸術と生活の境界を溶解させた国際的な前衛運動。

   イベントスコア（Event Score）：
   極めて短い「指示書」で構成される作品。

   ラ・モンテ・ヤングの例：
     "Draw a straight line and follow it."
     （直線を引き、それに従え。）

   オノ・ヨーコ『グレープフルーツ』（1964）：
     "Light a match and watch it till it goes out."
     （マッチに火をつけ、消えるまで見つめよ。）

   ジョージ・ブレクト「Word Event」（1961）：
     EXIT
     （退出）

   イベントスコアの構造：
   ・テキスト = 命令セット（プログラム）
   ・読者/演者 = インタプリタ（実行環境）
   ・実行結果 = パフォーマンス（出力）

   これはチューリングマシンの構造そのものである：
   ・遷移表 = イベントスコア
   ・TM = 演者の身体
   ・テープの変化 = 実行されたパフォーマンス

4. ライブコーディング — 即興的な実行

   ライブコーディング（Live Coding）：
   プログラムの作成と実行をリアルタイムで聴衆の前で行う
   パフォーマティブなプログラミング。

   ・TOPLAP（Temporary Organisation for the Pragmatics of
     Live Algorithm Programming, 2004〜）
   ・Alex McLean（Tidalcycles）、Renick Bell、
     Alexandra Cárdenas
   ・Algorave：アルゴリズムによるレイブ・イベント

   ライブコーディングの特徴：
   ・コードの「記述」と「実行」が同時に行われる
   ・スクリーンにコードが投影される（透明性）
   ・即興的：事前にすべてが決定されていない
   ・パフォーマーは人間のインタプリタであると同時に、
     コンピュータ上のインタプリタでもある

   即興詩との並行：
   ・即興詩人（口承詩の伝統、ラップ・フリースタイル）
   ・詩のスラム（Poetry Slam）
   ・いずれも「記述」と「実行」の同時性

   ライブコーディングは、チューリングマシンの
   「人間が計算を書き付ける行為」の現代的な再創造である。

5. 「インタプリタとしての身体」

   人間の身体は「テクストのインタプリタ」である：
   ・楽譜を読んで演奏する → 音楽のインタプリタ
   ・台本を読んで演じる → 演劇のインタプリタ
   ・レシピを読んで料理する → 料理のインタプリタ
   ・詩を読んで声に出す → 詩のインタプリタ

   チューリングの原論文が出発したのはまさにこの洞察：
   「人間が計算を行うとき、その行為の本質は何か」
   → 記号の読み取り、書き換え、移動
   → これを機械化したものがチューリングマシン

   しかし、人間のインタプリタには機械にないものがある：
   ・身体性（呼吸、疲労、感情）
   ・解釈の多義性（同じテクストの異なる実行）
   ・エラーの創造的利用（間違えることが表現になる）
   ・コンテクストへの感応（会場の雰囲気、聴衆の反応）

   機械のインタプリタ（コンパイラ・仮想マシン）は
   テクストを正確に実行するが、
   人間のインタプリタは「誤読」「逸脱」「即興」を通じて
   テクストを超えたものを生み出す。

ディスカッション・問い
================================================================

  1. シュヴィッタースの「ウルソナーテ」のテクストは
    「プログラム」か「楽譜」か「詩」か？
     あるいはその区別は意味がないか？

  2. フルクサスのイベントスコアを一つ選び、
     実際に「実行」してみよう。
     その体験はどのようなものだったか？
    同じスコアを別の人が実行したら、異なる結果になるか？

  3. ライブコーディングにおいて、「著者」は
     パフォーマーか、コンピュータか、アルゴリズムか、
     それとも三者の協働か？

  4. 自分のイベントスコア（3行以内の指示書）を
     書いてみよう。それは「実行可能」か？
     複数の「実行」が可能か？
